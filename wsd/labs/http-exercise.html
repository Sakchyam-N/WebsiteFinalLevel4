1. What is the Hypertext Transfer Protocol (HTTP)

-> Hypertext Transfer Protocol (HTTP) is an application-layer protocol for transmitting hypermedia documents, such as HTML. It was designed for communication between web browsers and web servers, but it can also be used for other purposes.

2. Why is HTTP called a stateless protocol?

-> HTTP is called as a stateless protocol because each request is executed independently, without any knowledge of the requests that were executed before it, which means once the transaction ends the connection between the browser and the server is also lost.

3. Describe the three phases of establishing a session using client-server protocols such as HTTP

-> (1)The client establishes a TCP connection (or the appropriate connection if the transport layer is not TCP).
(2)The client sends its request, and waits for the answer.
(3)The server processes the request, sending back its answer, providing a status code and appropriate data.

4. Describe the parameters on a get and post request.

-> GET : The parameter name and it's associated values are present.
POST: The keys and values are encoded in a data structure and non alphanumeric characters are percent encoded.

5. What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?

-> Response status codes indicate whether requests are successfully completed or not.
HTTP status code 200 - the request has succeeded.
HTTP status code 404 - the server was unable to find the requested resource.

6. What are HTTP cookies? Give an example.

-> Cookies are text files with small pieces of data — like a username and password — that are used to identify your computer as you use a computer network. Specific cookies known as HTTP cookies are used to identify specific users and improve your web browsing experience. Examples : Magic cookies

7. Why are cookies a security issue?

-> They can be read an modified/edited by those who have access to a clients hard disk in spite of the security measures taken to ensure cookies cannot fall into the hands of third parties and only go to the intended recipient.

8. Describe session hijacking and XSS

-> XSS, referring to cross site scripting is a security gap which allows an attacker to inject malicious client side code into the website. This code is later executed by the victims of the attack and allow the attacker to get past security/access controls and subsequently impersonate users. Session Hijacking relies on an attackers knowledge of a users session cookie. Once a user is authenticated on the server side, the attacker is capable of using the session cookie to hijack the session on their own browser.

9. Describe Cross-site request forgery(CSRF)->

-> An attacker can use XSS to launch a CSRF attack. The attacker causes a user's browser to send a request to the website without the user's own knowledge.

10. What are Cross-site HTTP requests (CORS) and what requests use CORS?

-> They are requests for resources belonging to a different domain than the domain of the resource that makes the request.
CSS Shapes from images makes such requests.
